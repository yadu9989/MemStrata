# Code of Conduct

This project follows the **Contributor Covenant v2.1**. The canonical text
lives at the URL below; we adopt it by reference rather than copying it
inline so updates to the upstream document apply here automatically.

**Canonical text:** https://www.contributor-covenant.org/version/2/1/code_of_conduct/

## Spirit

We want MemStrata to be a project where people can show up, do good
work, learn from each other, and disagree productively. That requires
treating each other as colleagues even when we've never met.

If you're contributing here, please:

- Assume good faith in others. Most disagreements are about technical
  trade-offs, not about character.
- Critique code, not people. "This approach has a problem because…" lands
  differently from "you don't understand X."
- Make room for newcomers. The first PR is the hardest one; help when
  you can.
- Keep discussions focused on the project. Off-topic threads can move
  to a different venue.

## Scope

The Code of Conduct applies to all project spaces: GitHub repository
interactions (issues, pull requests, discussions), commit messages,
official communication channels, and any in-person or virtual event
where someone is representing the project.

## Reporting

If you observe or experience conduct that doesn't fit the spirit above,
please contact the maintainers privately:

- **Email:** `conduct@memstrata.dev`
- **Subject line:** start with `[CoC]` so it's filtered correctly

For incidents that also have a security dimension, use
`security@memstrata.dev` instead (see [SECURITY.md](SECURITY.md)).

Reports are read by the project maintainers. We respond within 7 days
with an acknowledgment and a proposed next step. We keep reports
confidential by default; we'll never publish details without the
reporter's consent.

## Enforcement

Consequences follow the Contributor Covenant's enforcement ladder
(correction → warning → temporary ban → permanent ban). Which step
applies depends on severity and context. The maintainers make the
call, and we'll explain our reasoning to the parties involved.

We aim to be proportionate. A first-time misunderstanding gets a
conversation, not a ban. Repeated patterns or severe single incidents
get firmer responses.
